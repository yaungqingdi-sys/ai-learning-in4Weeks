# 项目理解提示词模板

```text
请帮我理解一个代码项目，但不要直接大改代码。

项目信息：
- 项目目标：[项目做什么]
- 我的水平：[例如 只能改小工具]
- 我现在最想弄清楚的问题：[例如 入口在哪、数据怎么流动]

请按下面顺序输出：
1. 这个项目的主要用途
2. 你推测的目录结构分工
3. 入口文件或关键文件
4. 一个最小执行流程：输入 -> 处理 -> 输出
5. 我现在最应该读的 3 个位置
6. 如果我要做最小改动，建议从哪里下手
7. 哪些地方你不确定，需要我补充上下文

要求：
- 不要假装看懂没给出的文件
- 明确区分“已知信息”和“推测信息”
- 如果建议改动，必须是最小改动思路
```
