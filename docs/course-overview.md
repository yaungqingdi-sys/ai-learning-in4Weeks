# 课程总览

本课程围绕一个核心问题展开：新人如何把 AI 从“会聊天的工具”变成“可验证、可复用、可迭代的学习与协作助手”。

## 当前结构

课程内容只保留 5 个部分：

- 目录：README 与课程总览
- 课程模块：核心概念与专题模块
- 课程考核：阶段测验、期末测试、回炉规则
- 学习 Prompt：可直接复用的学习提示词模板
- Reference：官方文档与高可信资料入口

## 课程模块

### 模块 1：AI 到底是什么

- 目标：搞清楚 AI、大模型、AI Chat、AI Search、Agent、AI Coding 的基本区别，并开始把 AI 当成一种工作中的思维方式。
- 产出：能用自己的话解释 AI 擅长什么、不擅长什么，也能判断哪些思考步骤适合先交给 AI 处理。

### 模块 2：如何和 AI 沟通

- 目标：理解提示词、上下文、元提示词和人机协作的基本结构。
- 产出：能写出一段稳定的学习型提示词。

### 模块 3：常用 AI 工具版图

- 目标：知道 ChatGPT、Google AI Studio、NotebookLM、Claude Code / Codex / OpenClaw 适合做什么。
- 产出：能按任务选工具，而不是按名气选工具。

### 模块 4：AI Coding 基础

- 目标：掌握终端、Git、代码理解、改小工具和验证的最小流程。
- 产出：能让 AI 帮你看懂项目、改一处小功能并验证结果。

### 模块 5：学习流程与模板固化

- 目标：理解一次性 Prompt 和流程模板的区别，能把稳定流程固化成模板。
- 产出：写出自己的学习 Prompt 组合和复盘模板。

### 模块 6：常见误区与边界

- 目标：识别新人最常见的学习误区，建立验证、责任和任务边界意识。
- 产出：能区分“看起来会了”和“真的学会了”，并能写出自己的使用边界清单。

### 模块 7：Vibe Coding 与 AI 原生开发

- 目标：理解 AI 原生开发里的上下文管理、规划驱动和逐步验证。
- 产出：能说清楚什么叫“先对齐，再执行，再验证”。

### 模块 8：使用 Gemini 绘制系统架构图

- 目标：学会在 Google AI Studio 中通过多轮澄清，把模糊想法逐步压实成可预览、可确认、可迭代的系统架构图。
- 产出：能让模型先提问、再出 ASCII 草图、再确认风格，最后生成可预览的 SVG。

### 模块 9：使用 Gemini 与 Codex 生成可控 PPT

- 目标：学会把 PPT 生成拆成“大纲确认、逐页确认、ASCII 布局、SVG 生成、可编辑 PPT 转换”五个阶段。
- 产出：能让 Gemini 先定内容和页面，再用 Codex skill 把已确认的 SVG 或 `.txt` 转成可编辑 `pptx`。

### 模块 10：人机互换

- 目标：学会在目标模糊、约束隐含、需要更高理解置信度时，把主动提问权暂时交给 AI，让它通过连续追问帮助人补齐信息。
- 产出：能设计一段“先测、先问、先重述、再执行”的对话流程，并能区分“AI 主导澄清”和“AI 替人做主”。

## 建议学习顺序

- 先学模块 1 到模块 3，建立 AI、工具与 Prompt 的基本地图。
- 再学模块 4 到模块 5，完成最小 AI Coding 协作与学习闭环搭建。
- 最后学模块 6 到模块 10，把误区、边界、AI 原生开发方法、可视化表达和 AI 主导澄清的方法串起来。
- 每完成一组模块，就做对应阶段测验；全部完成后做期末综合测试。

配套模板见 `../templates/`，外部资料入口见 [参考资料](reference.md)。

## 学习方式

- 每天 45 分钟。
- 一半时间学概念，一半时间做互动和产出。
- 每完成一个阶段至少做一次测验。
- 所有知识都尽量落到“解释出来”“做出来”“测出来”三个动作上。

## 学完后的最低能力线

- 能解释 AI 的基本概念、常见误区和使用边界。
- 能用 AI 帮自己拆学习任务、理解资料，并连接思路与工具。
- 能在小范围 coding 任务中与 AI 协作。
- 能把已确认的大纲和视觉稿转成可继续编辑的图示或 PPT。
- 能按照考核规则完成测验、错题回炉和复盘。
