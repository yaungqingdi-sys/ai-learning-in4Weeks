# 参考资料

- [Karpathy: Farzapedia 与个人 Wiki 式 AI 记忆](https://x.com/karpathy/status/2040572272944324650)：本模块的核心出发点，讲为什么个人知识库优于平台黑箱记忆——显式、可拥有、文件格式、AI 可替换。
- [Karpathy: LLM Knowledge Bases 工作流](https://x.com/karpathy/status/1934984795097678056)：Karpathy 关于用 LLM 构建个人知识库的完整工作流描述——数据摄入、Wiki 编译、Q&A、健康检查、工具扩展。
- [File over App — Steph Ango](https://stephango.com/file-over-app)：File over App 哲学的原始文章，解释为什么文件比应用更持久、更可控。
- [Obsidian 官网](https://obsidian.md)：Karpathy 用作 Wiki 浏览前端的工具，支持 Markdown 原生渲染、反向链接、插件扩展。
- [Obsidian Web Clipper](https://obsidian.md/clipper)：用来一键把网页文章转成本地 Markdown 文件的浏览器插件。
- [Vibe Coding 范式](../../module-7-vibe-coding/VibeCoding范式.md)：用来对照 Memory Bank 概念——Vibe Coding 里的记忆库和本模块的个人知识库在上下文管理上有直接关联。
- [搭建学习系统](../../module-5-build-learning-system/搭建学习系统.md)：用来对照学习流程模板化的思路——个人知识库可以看作学习系统的底层数据层。
- [全局参考资料](../../../reference.md)：用来继续查课程范围内的官方入口。
